# Atomadic Forge Conformity Statement CS-1 v1

## Overview

The Forge Conformity Statement CS-1 is a regulator-friendly artifact that bundles
EU AI Act Annex IV, Federal Reserve SR 11-7, FDA PCCP, and DoD CMMC-AI compliance
evidence into a single signed-or-signable document.

CS-1 is generated by `forge cs1` from a `ForgeReceiptV1` artifact.  It contains
no logic of its own — every claim is traceable to a specific Receipt field or
Lean4 corpus citation.

---

## Schema

**Schema identifier:** `atomadic-forge.cs1/v1`

### Top-level JSON shape

```json
{
  "schema_version": "atomadic-forge.cs1/v1",
  "generated_at_utc": "YYYY-MM-DDTHH:MM:SSZ",
  "receipt_schema_version": "atomadic-forge.receipt/v1",
  "receipt_generated_at_utc": "YYYY-MM-DDTHH:MM:SSZ",
  "project": { ... },
  "receipt_summary": { ... },
  "attestation": { ... },
  "compliance_claims": [ ... ],
  "regulator_questions": [ ... ],
  "lineage_chain_digest": "<sha256-hex or null>",
  "signatures_status": "SIGNED | PARTIAL | UNSIGNED",
  "notes": []
}
```

### Field definitions

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | string | Always `atomadic-forge.cs1/v1` |
| `generated_at_utc` | string | CS-1 emission timestamp (UTC ISO-8601) |
| `receipt_schema_version` | string | Schema version of the source Receipt |
| `receipt_generated_at_utc` | string | Timestamp of the source Receipt |
| `project` | object | Copy of `Receipt.project` block |
| `receipt_summary` | object | Compact Receipt summary (see below) |
| `attestation` | object | Lean4 attestation summary |
| `compliance_claims` | array | 17 compliance claims across 4 frameworks |
| `regulator_questions` | array | 5 standardized Q&A pairs |
| `lineage_chain_digest` | string\|null | SHA-256 of canonical lineage block |
| `signatures_status` | string | `SIGNED` / `PARTIAL` / `UNSIGNED` |
| `notes` | array | Free-form notes from the source Receipt |

### `receipt_summary` shape

```json
{
  "schema_version": "...",
  "generated_at_utc": "...",
  "forge_version": "...",
  "verdict": "PASS | FAIL | REFINE | QUARANTINE",
  "project_name": "...",
  "project_language": "python",
  "certify_score": 100.0,
  "wire_verdict": "PASS | FAIL",
  "wire_violation_count": 0,
  "symbol_count": 14
}
```

### `attestation` shape

```json
{
  "total_theorems": 567,
  "total_sorry": 0,
  "corpora_count": 2,
  "summary": "567 / 0 sorry",
  "corpora": [
    {
      "name": "aethel-nexus-proofs",
      "ref_sha": "deadbeef...",
      "theorem_count": 29,
      "sorry_count": 0
    }
  ]
}
```

### `compliance_claims` item shape

```json
{
  "framework": "EU AI Act",
  "ref": "Annex IV §1",
  "title": "General description of the AI system",
  "citation": "Regulation (EU) 2024/1689, Annex IV, paragraph 1",
  "receipt_field": "project.name + project.language + scout.symbol_count",
  "evidence": "..."
}
```

### `regulator_questions` item shape

```json
{
  "id": "RQ-1",
  "question": "What is the AI system and what does it do?",
  "answer_fields": "project.name, project.language, scout.symbol_count, ...",
  "framework_refs": "EU AI Act Annex IV §1; SR 11-7 §III.A"
}
```

## CLI Usage

### Generate CS-1 Markdown (default)

```bash
forge cs1 .
# writes .atomadic-forge/CS-1.md
```

### Generate CS-1 JSON

```bash
forge cs1 . --json
# prints JSON to stdout
```

### Custom receipt and output paths

```bash
forge cs1 /path/to/project \
  --receipt /path/to/receipt.json \
  --out /path/to/CS-1.md
```

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | CS-1 generated successfully |
| 1 | Receipt not found, unreadable, or missing required fields |

`UNSIGNED` receipts are **not** an error — CS-1 is generated and the
`signatures_status` field reflects the unsigned state.

## PDF Output

CS-1 is emitted as structured Markdown.  To produce a PDF:

```bash
# pandoc (recommended)
forge cs1 . && pandoc .atomadic-forge/CS-1.md -o CS-1.pdf

# wkhtmltopdf
forge cs1 . && pandoc .atomadic-forge/CS-1.md -t html | wkhtmltopdf - CS-1.pdf
```

No heavy PDF dependencies are required by Forge itself (stdlib only).
